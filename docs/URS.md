
# User Requirement Specification (URS)
**Project**: FindMyPark
**Version**: 1.0

## 1. Overview
This document captures the requirements from the perspective of the end-users (Drivers and Hosts).

## 2. Driver Requirements
- **URS-01**: As a driver, I want to find parking near my destination so that I don't waste time circling.
- **URS-02**: As a driver, I want to see the price before I book so that I can stay within budget.
- **URS-03**: As a driver, I want to filter for EV chargers because I drive an electric vehicle.
- **URS-04**: As a driver, I want to pay digitally so that I don't need cash.
- **URS-05**: As a driver, I want to see my past bookings for expense tracking.

## 3. Host Requirements
- **URS-06**: As a host, I want to list my unused driveway so that I can earn extra income.
- **URS-07**: As a host, I want to control when my spot is available.
- **URS-08**: As a host, I want to see how much money I have made.
- **URS-09**: As a host, I want to feel safe knowing who is parking (KYC).

## 4. Corporate/Admin Requirements
- **URS-10**: As an admin, I want to verify listings to ensure quality control.
- **URS-11**: As an admin, I want to see high-demand areas to target marketing.
